# Claude Design 實戰七招：Ryan Mather 使用心得完整翻譯

**來源：** Ryan Mather（Anthropic verticals team 設計師）X 推文串
**儲存日期：** 2026-04-18
**背景：** Claude Design 隨 Anthropic Labs 同步推出後，Ryan 分享了實戰 tips。他一人服務 7 條產品線，Claude Design 是撐住這個產量的關鍵工具。

---

## Tip 1：先把設計系統跟核心畫面設好

先花一小時把 design system（色彩、字型、按鈕樣式、間距、元件庫）跟核心畫面（登入、首頁、設定）餵給 Claude，後面每次生成都會在這基礎上延伸，視覺一致性自動保持，省掉逐張修圖的時間。

> 核心原則：第一步給的脈絡越明確，後面產出的品質越穩。

---

## Tip 2：跟工程師即時一起迭代

傳統設計流程「畫稿 → 評審 → 修改 → 交接 → 實作 → 對稿」要一兩週，新流程可以壓到一個會議。設計師跟工程師坐在一起，邊討論概念邊看 Claude 即時產出 mockup，討論層次停留在「這功能要解決什麼問題、有哪些約束」，而不是「這個按鈕放哪」。

> 協作粒度從「文件交接」變成「對話中即時定案」。

---

## Tip 3：用 Comment 工具做快速精準修改

粗稿出來後有幾十個細節想調，用嘴描述效率極低（「右上角那個按鈕再小一點、顏色淡一點...」）。直接用 Comment 工具點選元件做評註，資訊損耗幾乎是零。

> Point and crit（指著東西評論）從設計工作室的實體動作搬進 AI 介面。

---

## Tip 4：請 Claude 幫你的點子做影片 demo

Claude Design 不是 Figma 的升級版，它更接近 Claude Code 的使用體驗，是「跟會做東西的助手對話」的心智模型，而不是「在畫布上排版」。

> 當 Claude Code 的設計版來用，才能發揮它的完整能力（靜態稿、影片 demo、互動原型、客製小工具）。

---

## Tip 5：用 Connector（特別是 Docs 跟 Slack）

接上 connector 後可以下這樣的指令：「請讀一下產品檢討會議的筆記，做一份簡報探索裡面提到的所有問題的不同設計方案。」然後出去散步，回來驗收成果。

> 「整理會議結論 → 產出設計方向」從幾小時壓到幾分鐘，工作模式從「讀、想、做」變成「下指令、散步、驗收」。

---

## Tip 6：請 Claude 臨時做客製的小工具

Claude Design 能生產的不只是設計稿，而是「任何解決設計問題的東西」。例如設計色彩選擇器時，先請 Claude 做一個互動的色彩預覽 prototype 讓自己玩，邊玩邊調整規格，而不是憑空想像。

> 這在傳統設計軟體需要工程師配合，在 Claude Design 裡設計師一個人就能完成。
> 大膽實驗，你的設計速度會把過去的工作方式甩開好幾圈。

---

## Tip 7：知道什麼時候要慢下來、親手做

新圖示、小插畫、命名，這些細節永遠帶來不成比例的衝擊。Agentic 設計的超高速很誘人，但這三樣東西的決策空間極小、品質天花板極高——AI 能快速生出可用版本，但要做到「讓人記住」還是人在定奪。

> 知道什麼時候要降速親手做，本身就是一門藝術。這不是對 AI 工具的懷疑，是對「什麼適合自動化、什麼不適合」的清醒判斷。

---

## 結尾補充

- **設計愉悅感**：Claude Design 讓設計過程變得愉悅，可以嘗試更多發散的點子，對每個點子的執著也比較鬆。工具加速讓沉沒成本降低，決策更誠實。
- **Nate Parrott**：Claude Design 核心開發者，他的 whimsy（奇想）跟玩心帶進了整個產品，使用者感受得出來。
- **Ryan 的自嘲**：他自己也貢獻過幾個 PR（檔案畫面、composer 元件等），但那些程式碼現在全都被改掉了。
- **預告**：Claude Design 正式上線後會分享更多 tips 跟小技巧。

## 核心定位總結

| | 傳統設計工具（Figma） | Claude Design |
|--|---------------------|--------------|
| 心智模型 | 在畫布上排版 | 跟助手對話 |
| 產出物 | 設計稿 | 任何解決設計問題的東西 |
| 參照物 | 設計軟體 | Claude Code |
| 協作方式 | 文件交接 | 即時對話定案 |
| 自動化邊界 | 幾乎都要手動 | 除了圖示/插畫/命名以外都可以自動化 |

## What It Does

- **Overview** вҖ” Stats dashboard (doc count, word count, notes items) + recently updated docs + вӯҗ favorites
- **Library** вҖ” All documents grouped by folder, with inline markdown reader, AI summary, and вӯҗ favorite button
- **Categories** вҖ” Gemini AI auto-categorizes all notes and documents (persistent cache вҖ” survives restarts)
- **Search** вҖ” Full-text search across all notes and documents
- **Raw Notes** вҖ” Rendered markdown view of `notes.md`
- **Wiki** вҖ” Per-document AI summarization, concept ingestion, and cross-document knowledge synthesis
- **Concept Graph** вҖ” D3.js force-directed graph visualizing relationships between wiki concepts and documents
- **Lint** вҖ” AI health-check of notes quality with severity scoring and one-click fix
- **RWD** вҖ” Fully responsive; hamburger slide-in sidebar on mobile (вүӨ768px)

## Quick Start

### Option A: Docker (Recommended)

```bash
git clone https://github.com/Lewsiafat/MindVault.git
cd MindVault
cp .env.example .env
# Edit .env and add your GEMINI_API_KEY
docker compose up
```

Open http://localhost:8000/mind-vault/

### Option B: Manual

**Prerequisites:** Python 3.11+, Node.js 20+, [uv](https://docs.astral.sh/uv/)

```bash
git clone https://github.com/Lewsiafat/MindVault.git
cd MindVault

# Install dependencies
uv sync
cd frontend && npm install && npm run build && cd ..

# Configure
cp .env.example .env
# Edit .env and add your GEMINI_API_KEY

# Run
uv run uvicorn src.main:app --host 0.0.0.0 --port 8000
```

Open http://localhost:8000/mind-vault/

## Get a Gemini API Key

1. Go to [Google AI Studio](https://aistudio.google.com/)
2. Click **Get API Key** вҶ’ **Create API Key**
3. Copy the key into your `.env` file

The free tier (1,500 requests/day) is sufficient for personal use.

## Adding Your Content

Drop markdown files into the `data/` folder:

```
data/
в”ңв”Җв”Җ notes.md          # Your primary notes (parsed into sections)
в”ңв”Җв”Җ articles/         # Long-form articles and reading material
в”ңв”Җв”Җ saves/            # Saved references and links
```

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/notes` | Parse notes.md into sections + items |
| GET | `/api/library` | All docs with metadata, preview, mtime |
| GET | `/api/stats` | Total docs, words, notes items, folder counts |
| GET | `/api/doc?folder=&name=` | Full document content |
| GET | `/api/summary` | AI summary of entire knowledge base |
| GET | `/api/categorize` | AI categorization of all notes + docs |
| GET | `/api/doc-summary?folder=&name=` | AI summary of a single doc |
| GET | `/api/search?q=` | Full-text search across notes and docs |
| GET | `/api/wiki/status` | Wiki build status and pending docs |
| GET | `/api/wiki/pages` | All wiki pages |
| GET | `/api/wiki/graph` | Concept graph вҖ” `{nodes, edges}` |
| POST | `/api/wiki/ingest` | Ingest a document into wiki |
| POST | `/api/wiki/synthesize` | Cross-document knowledge synthesis |
| POST | `/api/lint` | Run AI lint health-check on notes |
| POST | `/api/lint/fix` | Auto-fix a single lint issue |
| GET | `/api/health` | Health check |

## Deployment

### VPS with nginx

```nginx
location /mind-vault/ {
    proxy_pass http://127.0.0.1:8000/mind-vault/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
}
```

### Systemd service

```ini
[Unit]
Description=MindVault
After=network.target

[Service]
User=youruser
WorkingDirectory=/path/to/MindVault
ExecStart=/path/to/.venv/bin/uvicorn src.main:app --host 127.0.0.1 --port 8000
EnvironmentFile=/path/to/MindVault/.env
Restart=always

[Install]
WantedBy=multi-user.target
```
